Lecture 11 — Entity Relationship Diagrams and System Architecture

  • This lecture helped me understand how system analysis moves from abstract ideas to concrete database design.
  • We recapped how use cases can be transformed into flowcharts and then into actual pages within a system.
  • The role of the database administrator became clearer, especially how ERDs guide the creation of tables that support application pages.
  • I also gained a clearer view of system architecture, where the frontend focuses on user interface and logic, the backend manages the database, and the middle layer connects them through APIs and web services.
